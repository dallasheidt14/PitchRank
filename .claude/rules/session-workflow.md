# Session Workflow

## After implementing changes
Always run `/finalize` after code changes are complete. This runs tests, polishes code, reviews, and commits — the quality gate before anything ships.

## Before ending a session
Always run `/self-improve` before wrapping up. This captures learnings from the session so they're not lost when context is cleared.

## During long waits
When a ranking run, CI pipeline, or deploy is in progress, suggest productive work the user can tackle in parallel rather than sitting idle. Examples: review open PRs, run data hygiene scripts, write tests, update docs.
